# Developer Advertising Case Study

## Why These Ads Work

Great developer ads show real work.

They do not say:

> AI helps you code.

They say:

> Run this workflow and get this result.

## Pattern

Developer problem → command/action → artifact → shipped outcome

## Common Transformations

- Missing tests → generated coverage → reviewable PR
- Stack trace → fix → passing tests
- Backlog item → implementation → PR
- Plan → parallel attempts → selected result

## Copy Principles

- Use verbs.
- Show the interface.
- Compress the workflow.
- End with a concrete artifact.
- Keep language literal.
- Avoid generic AI claims.

## Example Headline Patterns

- Turn TODOs into pull requests
- Turn stack traces into fixes
- Start in the CLI. Finish in a PR.
- Run parallel agents. Ship one result.
- Go from plan to pull request.

## Agent Instructions

When generating ads:

1. Start with a real developer task.
2. Use a concrete action.
3. Show the artifact or output.
4. Keep copy tight.
5. Avoid abstract positioning.
