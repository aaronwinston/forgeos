# Workflow Narrative Case Study

## Core Thesis

The value is not in one surface. It is in the flow between them.

## Model

Operate → Craft → Ship

- **Operate** — fast execution and proof of value
- **Craft** — refinement, debugging, and trust
- **Ship** — durability, review, collaboration, and governance

## Narrative Pattern

1. Start with a task.
2. Execute quickly.
3. Refine where correctness matters.
4. Ship through a durable review surface.
5. Show that the whole system reinforced the workflow.

## Strategic Insight

A surface can create attention. A connected workflow creates adoption.

## Campaign / Demo Pattern

1. Start from an idea or issue.
2. Generate a concrete change.
3. Inspect or refine the change.
4. Open a reviewable artifact.
5. Ship or reject with confidence.

## Agent Instructions

Use this file when a story must connect multiple product surfaces, channels, or stages of work. Do not use it for isolated feature announcements.
