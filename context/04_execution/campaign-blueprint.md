# Campaign Blueprint

## Core Idea

Drive adoption through workflow-based storytelling.

## Campaign Structure

Problem → Workflow → Proof → Outcome

## Objectives

A strong campaign should:

- Drive awareness and usage
- Shift perception from feature to workflow system
- Create a bridge from curiosity to activation
- Help the audience understand what to try next

## Audience

Developers want speed, clarity, reliability, and workflows that respect how they already work.

## Key Strategy

- Show real workflows.
- Make the interface visible.
- Use concrete developer actions.
- Bridge surfaces when possible.
- Avoid abstract AI messaging.

## Creative Rule

The product proof should be visible.

Examples of good proof artifacts:

- CLI command
- diff
- PR
- test result
- log output
- issue
- review comment
- workflow diagram

## Agent Instructions

When creating a campaign:

1. Name the workflow.
2. Identify the developer problem.
3. Show the action.
4. Show the artifact.
5. Show the outcome.
6. Build channel-specific variants from the same core story.
