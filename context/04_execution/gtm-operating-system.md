# GTM Operating System

## Core Idea

One workflow demo should become many assets.

## Operating Cadence

### Weekly

- Review upcoming launches
- Identify workflow value
- Check demo readiness
- Resolve blockers

### Monthly

- Review performance
- Analyze activation
- Review cross-channel learnings

### Quarterly

- Refresh narrative
- Prioritize workflows
- Plan major moments

## Launch Framework

### Step 1: Intake

Clarify product truth, audience, lifecycle stage, and workflow value.

### Step 2: Core Narrative

Create one clear narrative that every channel can reuse.

### Step 3: Demo Development

The demo is the product. If the demo is weak, the story will be weak.

### Step 4: Derivative Content Factory

One demo can produce:

- Blog post
- Video
- Docs tutorial
- Social posts
- Email
- Enablement card
- Landing page section
- In-product CTA

### Step 5: Distribution

Distribute through the channels that match audience intent and lifecycle stage.

### Step 6: Measurement

Measure activation, workflow completion, surface expansion, and qualitative feedback.

## Risk Mitigation

- Developer mistrust → show real workflows, no hype
- Competitor velocity → create faster derivative cycles
- Launch inconsistency → standardize intake and narrative
- Team overload → turn one demo into many assets

## Agent Instructions

Never generate a single asset in isolation when a campaign system is appropriate. Always ask what the demo can become.
