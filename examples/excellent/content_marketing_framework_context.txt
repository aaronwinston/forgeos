# GitHub Content Marketing Framework FY25 (Agent Context)

## 🔑 How to Use This Document (Agent Notes)

- This is a **strategic content framework**, not messaging or copy.
- Use it to **align content to narrative, audience, and pillars**.
- It defines the **why and direction**, not the execution.
- Always pair this with:
  - Messaging/positioning docs (for what to say)
  - Execution frameworks (for what to produce)

### What this is best for
- Choosing a **content pillar** (Speed, Scale, Security)
- Selecting a **target audience/persona**
- Ensuring **narrative consistency across outputs**

### What NOT to use it for
- Writing copy directly
- Making product claims
- Acting as a source of truth for features

### Key rule
This document is a **constraint system** — not a generator.

---

## 🧠 Core Framework

### North Star
Developer expertise matters more than ever in the age of AI.

### Key Message
Software development is changing—fast.

### Positioning
GitHub defines the future of software development through better workflows: faster, collaborative, secure.
